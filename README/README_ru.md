## 📖 Обзор

LogIO — это современная библиотека логирования промышленного уровня для языка C. Она предоставляет **асинхронную запись**, **автоматическую ротацию файлов**, **структурированные JSON‑логи**, **поддержку множества выводов** и **нулевые накладные расходы при отключении на этапе компиляции** — всё через простой API.

Ключевые возможности:

- 🔀 **Полностью асинхронная** – выделенный поток собирает и записывает логи, вызывающий код не блокируется вводом-выводом.
- 🔒 **Потокобезопасность** – внутреннее состояние защищено мьютексом, безопасно в многопоточной среде.
- 📁 **Автоматическое управление файлами** – создаёт дерево каталогов и имена файлов с метками времени (например, `%Y-%M-%D_%h:%m:%s.log`).
- 📊 **Структурированные логи** – опциональный вывод в JSON с экранированием, готов для ELK / Loki.
- 🎨 **Цвета в консоли** – опциональные ANSI‑цвета (DEBUG — голубой, WARN — жёлтый, ERROR — красный) при выводе в терминал.
- 🔄 **Ротация файлов** – по размеру или по времени, старые файлы автоматически переименовываются.
- 🎯 **Уровни логирования** – DEBUG, INFO, WARN, ERROR, порог настраивается во время выполнения.
- 🔌 **Расширяемость** – обратные вызовы или дополнительные потоки `FILE*` для собственных приёмников (WebSocket, UDP…).
- ⚡ **Отключение без накладных расходов** – `#define LOG_ENABLED` полностью удаляет код логирования при компиляции.

## ✨ Матрица функций

| Функция | Описание |
|---------|----------|
| Асинхронный ввод-вывод | Очередь + выделенный поток записи, `LogPrintf` не блокирует |
| Потокобезопасность | Мьютекс + условная переменная для защиты очереди и состояния |
| Ротация | По размеру (например, 10 МБ) или по времени (например, каждый час) |

| Цветной вывод | Автоопределение терминала, ANSI‑цвета |
| Отключение компилятором | `#define LOG_ENABLED` полностью исключает код логирования |
| Обратные вызовы | Каждое сообщение может быть обработано пользовательской функцией |

## 🚀 Быстрый старт

### Требования

- Компилятор C99 (GCC или Clang)
- Библиотека POSIX threads (`pthread`)
- CMake ≥ 3.5 или обычный Make

### Установка

```bash
git clone https://github.com/Lemonade-NingYou/logio.git
cd logio
make
sudo make install
```

В Termux:
```bash
make install PREFIX=/data/data/com.termux/files/usr
```

### Минимальный пример

```c
#include "logio.h"
#include <stdlib.h>

int main(void) {
    // 1. Инициализация – автоматически создаст ./logs/app_2026-06-19_14:30:00.log
    if (InitLog("./logs/app_%Y-%M-%D_%h:%m:%s.log", LOG_LEVEL_DEBUG) != 0) {
        return 1;
    }

    // 2. Одновременно выводим в консоль с цветами
    LogAddOutputStream(stdout, 1);

    // 3. Текстовый лог
    LogPrintf(LOG_LEVEL_INFO, "Сервер запущен на порту %d", 8080);
    LogPrintf(LOG_LEVEL_DEBUG, "Значение конфигурации: x=%d", 42);

    // 4. JSON‑лог
    LogPrintfJSON(LOG_LEVEL_INFO, "Администратор admin вошёл в систему");

    // 5. Включаем ротацию по размеру (новый файл каждые 5 МБ)
    LogSetRolling(LOG_ROLL_SIZE, 5, 0);

    // 6. Принудительная запись перед выходом
    LogFlush();
    return 0;
}
```

Компиляция и запуск:
```bash
gcc -std=c99 -pthread -o example example.c logio.c
./example
```

После запуска в каталоге `logs/` появится файл вида `app_2026-06-19_14:30:00.log`, а в консоли будут цветные строки.

## 📚 API

### Инициализация
```c
int InitLog(const char *logFilePath, LogLevel level);
```
- `logFilePath` – путь с форматом времени (`%Y` – год, `%M` – месяц, `%D` – день, `%h` – час, `%m` – минута, `%s` – секунда, `%%` – литерал `%`).  
  `%N` разворачивается в формат по умолчанию `%Y-%M-%D_%h:%m:%s`.
- `level` – порог (0=DEBUG, 1=INFO, 2=WARN, 3=ERROR).  
- Возвращает `0` при успехе, `-1` при ошибке (в случае ошибки вывод перенаправляется в `stderr`).

### Текстовый лог
```c
void LogPrintf(LogLevel level, const char *fmt, ...);
```
Формат аналогичен `printf`, автоматически добавляется префикс `[УРОВЕНЬ/ВРЕМЯ]`.

### JSON‑лог
```c
void LogPrintfJSON(LogLevel level, const char *fmt, ...);
```
Выводит одну строку JSON:
```json
{"level":"INFO","time":"2026-06-19 14:30:00","msg":"Ваше сообщение"}
```
Строка сообщения экранируется. Все выходные потоки и колбэки получают эту JSON‑строку.

### Управление выводами
```c
int LogAddOutputStream(FILE *stream, int enable_color);
```
Добавляет поток (например, `stdout`, `stderr`). Если `enable_color` не равен нулю и поток является терминалом, включаются ANSI‑цвета.  
Возвращает идентификатор вывода (≥0) или `-1` при ошибке.

```c
int LogAddCallback(LogCallback cb, void *userdata);
```
Регистрирует пользовательский обратный вызов, который будет вызываться для каждого сообщения.

```c
typedef void (*LogCallback)(LogLevel level, const char *message,
                            time_t timestamp, int is_json, void *userdata);
```
- `level` – уровень логирования.
- `message` – текст сообщения (если `is_json` не ноль, это уже экранированная JSON‑строка).
- `timestamp` – временная метка Unix.
- `is_json` – ненулевое значение, если сообщение было отправлено через `LogPrintfJSON`.
- `userdata` – пользовательские данные, переданные при регистрации.

```c
int LogRemoveOutput(int id);
```
Удаляет ранее добавленный вывод по идентификатору. Возвращает `0` при успехе.

### Ротация файлов
```c
void LogSetRolling(LogRollMode mode, long max_size_mb, int time_interval_sec);
```
- `LOG_ROLL_NONE` – без ротации (по умолчанию).
- `LOG_ROLL_SIZE` – ротация по размеру: новый файл создаётся, когда текущий превышает `max_size_mb` МБ.
- `LOG_ROLL_TIME` – ротация по времени: новый файл создаётся каждые `time_interval_sec` секунд.

При ротации текущий файл переименовывается (добавляется временная метка) и открывается новый.

### Сброс (принудительная запись)
```c
void LogFlush(void);
```
Блокирует вызывающий поток до тех пор, пока асинхронная очередь не опустеет и все данные не будут записаны на диск. Рекомендуется вызывать перед завершением программы или после критических операций.

### Отключение на этапе компиляции
Определите макрос `LOG_ENABLED` **перед** включением `logio.h`, чтобы полностью удалить код логирования:

```c
#define LOG_ENABLED
#include "logio.h"
```

После этого все вызовы логирования превращаются в пустые операции, не увеличивая размер двоичного файла и не создавая накладных расходов. Рекомендуется для релизных сборок.

## 🎯 Продвинутое использование

### Пример колбэка – пересылка в WebSocket

```c
void ws_forward(LogLevel level, const char *msg, time_t ts, int is_json, void *ws) {
    if (is_json) {
        websocket_send(ws, msg);          // уже JSON
    } else {
        char buf[512];
        snprintf(buf, sizeof(buf),
                 "{\"level\":%d,\"time\":%lld,\"msg\":\"%s\"}",
                 level, (long long)ts, msg);
        websocket_send(ws, buf);
    }
}

// В main:
void *ws_conn = websocket_connect("ws://logserver:9000");
LogAddCallback(ws_forward, ws_conn);
```

### Ротация по времени

```c
// Новый файл каждый час
LogSetRolling(LOG_ROLL_TIME, 0, 3600);
```

### Многопоточное использование

```c
#include <pthread.h>

void* worker_thread(void* arg) {
    LogPrintf(LOG_LEVEL_INFO, "Рабочий поток %ld запущен", (long)arg);
    // ... полезная работа
    LogPrintf(LOG_LEVEL_INFO, "Рабочий поток %ld завершён", (long)arg);
    return NULL;
}

int main() {
    InitLog("./logs/app.log", LOG_LEVEL_DEBUG);
    pthread_t threads[5];
    for (int i = 0; i < 5; i++)
        pthread_create(&threads[i], NULL, worker_thread, (void*)(long)i);
    for (int i = 0; i < 5; i++)
        pthread_join(threads[i], NULL);
    LogFlush();
    return 0;
}
```

## 🐛 Часто задаваемые вопросы

**1. Ошибка компиляции: не найден `pthread_create`**  
При линковке добавьте `-lpthread` в конце команды:
```bash
gcc ... -lpthread
```

**2. Не удаётся создать каталог для логов**  
Убедитесь, что родительский каталог существует, и у вас есть права на запись. `InitLog` автоматически создаёт промежуточные каталоги, но недостаток прав на корневой каталог может привести к ошибке.

**3. Логи не появляются сразу**  
LogIO использует асинхронную очередь. Вызовите `LogFlush()` для принудительной записи. При штатном завершении программы буферы также сбрасываются.

**4. `vsnprintf` не объявлен**  
Используйте `-std=c99` или определите `_POSIX_C_SOURCE=200112L` перед включением заголовочных файлов, чтобы включить расширения POSIX.
